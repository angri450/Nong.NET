Angri450.Nong.Tool.Chart 25.87 MB NEW

独立 dotnet tool，命令名 nong-chart。通过 nong chart ... 触发。

源模块作用
ChartCore统计图表核心（ScottPlot 绑定）
ThirdParty含 ScottPlot + SkiaSharp + HarfBuzzSharp 源码（编译时合并）
SkiaSharp.NativeAssets.Win32Windows 渲染 DLL (NuGet)
HarfBuzzSharp.NativeAssets.Win32Windows 文字塑形 DLL (NuGet)

命令 (11): bar / line / scatter / pie / boxplot / histogram / heatmap / radar / analyze / anova / duncan

瘦身前 83MB。删掉 macOS/Linux 的 SkiaSharp/HarfBuzz native 库。

Angri450.Nong.Tool.Diagram 25.86 MB NEW

独立 dotnet tool，命令名 nong-diagram。

Angri450.Nong.Tool.Pdf 28.68 MB NEW

独立 dotnet tool，命令名 nong-pdf。含多平台 PDFium native DLL。

源模块作用

命令 (8): check / dissect / render / images / merge / split / ocr / compress

pdf ocr 通过子进程调用 nong-ocr，不内嵌 OCR 引擎。

Angri450.Nong.Tool.Ocr 12.14 MB NEW

入口: nong word images --analyze / --crop / nong word crop 转发到 nong-imaging

功能有损失吗？

模块命令数拆分前拆分后变动
word39CLI 内嵌CLI 内嵌无变化
chart11CLI 内嵌 SkiaSharpnong-chart 外部调用，同一份命令源码命令不变
diagram3CLI 内嵌 MSAGLnong-diagram 外部调用命令不变
pdf8CLI 内嵌 PDFiumnong-pdf 外部调用命令不变
pptx4CLI 内嵌nong-pptx 外部调用命令不变
ocr11CLI 内嵌 PaddleOCRnong-ocr 外部调用命令不变
imaging4CLI 内嵌 SkiaSharpnong-imaging 外部调用命令不变
excel/inspect/lit/slice/genre/icons/skill/progress45CLI 内嵌CLI 内嵌无变化

每个外部工具的 csproj 用 <Compile Include="..\..\Cli\Commands\XxxCommands.cs" /> 编译同一个源码文件。逻辑一行没改，只是编译目标变了。

已知弱化 -- chart/diagram/imaging 只含 Windows native DLL。Linux/macOS 用户不能直接用 dotnet tool install，需从源码构建。这是为了包体积从 83MB 压到 26MB 的有意取舍。
