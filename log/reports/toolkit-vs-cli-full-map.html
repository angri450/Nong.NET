1. 三层架构

Nong.NanoBot.Net（Agent 运行层）
  Nanobot.Core/Agent、WebUI、Channel、Memory、MCP、插件/技能启动
  Nanobot.Web -- 浏览器工作台
  内置 SkillLoader 加载 Nong.Toolkit.Net 技能

Nong.Toolkit.Net（技能教学层）
  15 个 SKILL.md 文件，作为 Claude Code 技能包
  word/pdf/literature/inspect/excel/chart/diagram/pptx/multimodal/genre/icons
  slice/skill/skill-manager/progress-report
  不放 C# 执行逻辑，只放文档和少量 PowerShell 辅助脚本

Nong.Cli.Net（确定性执行层）
  96 个命令，15 个命令模块，一个 nong.dll
  文档、图表、OCR 等操作都通过 nong 进程调用
  ThirdParty.csproj 合并 15 组第三方源码目录
  SkillManagerCore 提供 skill validate/scan/inventory/package

5. 不对齐问题状态

P0 - 版本过期

已修复 Toolkit 现在通过共享预检文档统一要求 Nong.Cli.Net 4.0.0+。

以后版本号只改 references/shared/nong-cli-preflight.md，不要散落在每个 SKILL.md。

P1 - progress-report 放错位置

已修复 CLI 仓库里误放的 progress-report skill/tool 已撤回，Toolkit 只保留技能指导。

实际生成报告走 nong progress report --project-root . --json。

P2 - Toolkit 日志不成形

已修复 Toolkit 的 log/ 已整理成 plans、changelog、debug、guidance 和索引。

后续工作继续写进结构化日志目录。

P3 - 预检安装块重复

已修复 重复的 SKILL.md 安装说明已经改成共享预检引用。

不要再把安装块复制回各个技能文件。

P4 - skill-manager 孤立

已修复 skill-manager 已纳入 Toolkit，并登记到插件 manifest。

真正的打包闸门仍用 nong skill validate/scan/inventory/package。

P5 - Toolkit 缺少开发指导

已修复 Toolkit 已有 CLAUDE.md，写清仓库身份、边界、工作流和验证规则。

确定性执行逻辑继续放在 Nong.Cli.Net，不放进 Toolkit。

P6 - CLI 有命令但 skill 没有

已修复 slice 和 skill 技能已覆盖对应 8 个命令。

以后 CLI 命令面变化后，同步验证这两个技能。

P7 - 部分技能太薄

已修复 pdf、literature、inspect 已补聚焦 reference。

reference 保持短而准，需要时再加载。

P8 - NanoBot 文档旧名字

基本修复 活跃 NanoBot 文档现在使用 Nong.NanoBot.Net、Nong.Toolkit.Net、Nong.Cli.Net 命名；历史 changelog 只保留历史语境。

包名和历史记录可以保留旧语境，但 active docs、命令和路径不能再用旧名。

P9 - NuGet lock 文件过期

已修复 已用 dotnet restore --use-lock-file --force-evaluate 重算 lock 文件；旧 Nong 3.2.x/3.1.x/0.1.x 依赖范围已清掉。

不要手改 lock 文件。项目引用包元数据变动时，重新跑 force-evaluate restore。

P10 - progress-report 命令缺失

已修复 Nong.Cli.Net 已暴露 nong progress report。

回归测试覆盖 JSON 输出和 HTML 报告生成。

P11 - 公文底层能力没暴露到 CLI

已修复 GongWenFormatter 和 OfficialDocWriter 已通过 nong word format-gongwen 与 nong inspect write-official 暴露。

Toolkit 的 word/inspect/genre 现在都会把公文草稿和公文排版路由到已实现 CLI 命令。

P12 - 全局 nong shim 坏了

已修复 旧全局 shim 指向不存在的 3.2.4 tool-store 路径。已备份旧 shim，并安装本地打包的 Nong.Cli.Net 4.0.0 tool。

nong --version 现在返回 4.0.0，nong commands --json 返回 96 个命令。
