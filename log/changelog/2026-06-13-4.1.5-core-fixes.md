# v4.1.5: 三个核心修复 + 两个次要修复

Date: 2026-06-13
相关 debug: `log/debug/2026-06-13-core-issues-round1.md`

## 修复清单

### 1. OCR preflight 去阻塞化 (critical)

- `LooksLikeQrOrCodeGraphic` + `LooksLikeGraphicOnlyImage` 从 blocking 改为 warning
- 启发式检测结果标记 Classification 但不设 ShouldSkip
- CLI handler 打印 warning 后自动继续推理，仅 ZXing 解码成功的条码保持 blocking
- 阈值收紧以减少误判

### 2. word fill 表格位置填充 (critical)

- 新增 `FillTablesByPosition`: JSON key `"tables"` 触发
- Schema: `{"tables": [{"index": 0, "rows": [{"cells": ["A1", "B1"]}]}]}`
- 模板无需 {{tag}} 占位符，按表格序号+行列号直接填值
- 兼容原标签流程：tables 先处理→移除→剩余走占位符替换

### 3. NongMark 字体属性 (critical)

- `::: paragraph {font="仿宋_GB2312" fontAscii="Arial" size=16}` 语法
- `ApplyParagraphAttrs` 解析 font/size/bold 属性
- `MakeRun` 从 static 改为实例方法，读取当前字体状态
- 段落结束 `ResetParagraphState` 恢复默认值

### 4. OCR 日志噪音抑制 (minor)

- `ConfigureNativeLogEnvironment` 移至静态构造函数（类型加载时）
- 新增 `GLOG_v=0`、`FLAGS_v=0`、`PADDLE_DISABLE_SIGNAL_HANDLER=1`

### 5. check-env 版本号修正 (minor)

- `CliVersion.Current` 从 "4.1.2" → "4.1.5"

## 影响范围

- `Angri450.Nong.Tool.Ocr` → 4.1.5
- `CliVersion.Current` 升级影响所有共享此常量的工具
- `Angri450.Nong.Docx` 库新增 `FillTablesByPosition` / `SetCellText` API

## 未修复 (留待后续)

- PDF 图片解码失败 (20/24 张线框图无法提取) — 需 PDF 模块专项优化
