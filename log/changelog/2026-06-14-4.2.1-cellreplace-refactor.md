# v4.2.1: Word 模板填充引擎重构 — cellReplace + tableRows

Date: 2026-06-14

## 为什么重构

`FillTablesByPosition` 经历了反复修补——用数字 index 定位表，但 `Descendants<Table>()` 递归进了嵌套表格，表序和 `word read` 对不上；改成 `Elements<Table>()` + `match` 文字匹配后，又陷入"标签不保留 / 多段落结构被删 / 换行符不转 / 行高不动"的交替修复。

根本问题不在实现细节，在架构假设——按表的物理行号/列号来定位，需要调用者精确知道每张表有几行几列、哪些格是值格哪些是标签格。这等于把表格布局知识外化到了 JSON 里。

## cellReplace：按键值匹配

新引擎 **不再关心表序、行号、列号**。原理：

1. 遍历文档中所有表格的所有单元格
2. 对每个单元格，取 `InnerText.Trim()`，在 `cellReplace` 字典中找 key（`StartsWith` 匹配）
3. 匹配成功后：
   - 该单元格**右方有格子** → 填右格（适合"标签 | 值"两列表格）
   - 该单元格**右方无格子** → 填本格（适合单格描述表）
4. 匹配的 key 填空字符串 `""` → 保留标签原样不覆盖

**对比**：

| | FillTablesByPosition | cellReplace |
|---|---|---|
| 表定位 | 数字 index（易漂移） | 单元格文字，无漂移 |
| JSON 格式 | `{"index":8,"rows":[{"cells":[...]}]}` | `{"单元格文字":"值"}` |
| 未填充的表 | 模板文字泄漏（需手动清空） | 原封保留（承诺书、学院意见） |
| 标签保留 | 需额外传 `""` | key 为 `""` 时跳过 |

## tableRows：追加数据行

处理带表头行的数据表（如专利清单）。原理：

1. 用 key 匹配表格 `InnerText`
2. 保留第 0 行（表头格式），删除其余行
3. 按提供的二维数组逐行逐列填入，继承表头行的字体和行属性

```json
{
  "tableRows": {
    "专利号/登记号": [
      ["实用新型专利", "一种xxx装置", "ZL...", "2024-01-01"]
    ]
  }
}
```

## 附加改进

- **固定行高 → 自动行高**：`ReplaceCellsByText` 遍历表格时清除所有 `w:trHeight`，长文本自动撑开不裁切
- **`\n` → Word 换行符**：`FillCellContent` 将 JSON 中的 `\n` 转成 `<w:br/>`
- **字体保留**：`FillCellContent` 在删除原有段落之前先读取 `RunFonts` 和 `FontSize`，补回新段落，保持文件风格一致

原有 `{{tag}}` 占位符 + `@if/@foreach` 模式不变，`cellReplace` 在占位符处理之前执行。
