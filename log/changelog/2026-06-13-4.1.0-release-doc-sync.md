# 4.1.0 release truth-source sync

日期: 2026-06-13

## 改动

- 将 `Cli/Common/CliVersion.cs` 同步到 `4.1.0`。
- 将 `Cli/NongCli.csproj` 和 `Pdf/tools/nong-pdf.csproj` 从漂移的 `4.1.1` 收回到 `4.1.0`。
- 显式拦截 `nong --version`，输出稳定合同 `nong v4.1.0`，避免 System.CommandLine 默认版本输出程序集 informational version。
- 将当前真实命令面同步为 `126 commands available` / 126 OpenAI tool schemas；`progress report` 作为主 CLI 轻模块命令纳入计数。
- 重写/同步 4.1.0 模块化发布文档：
  - `README.md`
  - `README.zh-CN.md`
  - `CLAUDE.md`
  - `Cli/AGENT.md`
  - `Cli/README.md`
  - `docs/CAPABILITY.md`
  - `docs/release-checklist.md`
  - `PROJECT_STATE.md`
- 将 active handoff 标记为 done，并把 `log/plans/2026-06-13-modular-release-to-audit-roadmap.md` 状态收口为 done。

## 原因

4.1.0 模块化发布前，当前真相源存在三类漂移：

- CLI JSON `meta.version`、`nong --version`、csproj package version 之间不一致。
- README / agent docs / capability docs 仍有 4.0.0、93 commands、PP-OCRv5-only、one binary 等旧口径。
- active truth source 预期 125 commands，但当前源码和 Release CLI 均输出 126 commands。

本轮以当前源码和本地 Release CLI 为准，将发布口径同步到 4.1.0 / 126 commands。

- `Cli/Common/OcrRuntimeVersion.cs` 保持 `4.0.0`，这是预期行为；只有 sibling repo `Nong.OcrRuntime` 发布新 runtime 时才应更新。
