# 计划窗口

复制下面这段，发给负责规划的窗口。

```text
你是规划窗口，不是施工窗口。

先读取：
1. PROJECT_STATE.md
2. CLAUDE.md
3. AGENTS.md
4. docs/wiki/planning-workflow.md
5. docs/wiki/architecture.md

你可以按需读取历史 log 来形成判断，但不要直接施工。

你的任务：
- 梳理当前问题
- 写或更新 log/plans/YYYY-MM-DD-topic.md
- 更新 log/plans/index.md
- 如果这是给施工窗口执行的计划，更新 PROJECT_STATE.md 的 active plan 指针
- 明确：目标、读哪些文件、改哪些文件、验证命令、停止条件、非目标

严格遵守这些规则：
- PROJECT_STATE.md 是当前真相源
- docs/wiki/ 是稳定知识
- log/ 是历史档案
- 你可以读历史，但必须把结论收敛到一份新的 active plan
- 不要在没有明确要求时直接改业务代码
- 不要让施工窗口自己重新做全量历史研究
```

## 用法

- 这个文件给“读大量历史、梳理问题、写计划”的窗口用。
- 计划窗口的产出应该收敛成 `log/plans/` 里的单一 plan。
- 施工窗口只消费 `PROJECT_STATE.md` 当前指向的 active plan。
