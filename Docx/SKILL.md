# Word 模板填充 (cellReplace)

## 适用场景

用户有一个 DOCX 模板（无 {{tag}} 占位符），需按模板表格填入数据生成申报书、验收报告、合同等。模板表格有两种：

1. **标签-值表**：两列或多列，第一列是标签（"项目名称"、"联系人"），第二列是值格（空白，待填）
2. **单格描述表**：一整格，包含提示文字如"（描述项目定位、应用领域……300字以内）"

## 命令

```bash
nong word fill 模板.docx 数据.json -o 输出.docx --json
```

## 数据 JSON 格式

### cellReplace（主模式）

键 = 模板单元格中的文字（StartsWith 匹配），值 = 要填入的内容。引擎自动判断填法：

```json
{
  "cellReplace": {
    "项目名称": "基于xxx技术的xxx系统",
    "项目主申报人": "张三",
    "联系人": "张三",
    "联系电话": "13800000000",
    "邮箱": "zhangsan@xxx.edu.cn",
    "项目所属领域": "■ 现代种业与生物育种",
    "项目是否成立公司": "■ 否",
    "公司名称": "--（暂未成立公司）",
    "注册地点": "--",
    "成立时间": "--",
    "（描述项目定位": "本项目提供了一种……（详细描述）",
    "（阐述项目目前": "本项目已获……（详细描述）",
    "（请对市场需求": "随着……（详细描述）",
    "（请说明项目当前": "拟开展概念验证……（详细描述）",
    "（基于以上内容": "1. 技术创新……\n2. 应用前景……\n3. 转化潜力……",
    "姓 名": "张三",
    "年 龄": "35",
    "证件类型": "身份证",
    "身份证号": "000000000000000000",
    "最终毕业院校": "某某大学",
    "最高学历": "博士",
    "专 业": "农业工程",
    "职 称": "教授",
    "联系方式": "13800000000",
    "邮 箱": "zhangsan@xxx.edu.cn",
    "（填写内容应实事求是": "张三，教授，长期从事……（详细描述）",
    "（介绍核心团队": "项目核心团队共5人……（详细描述）",
    "（如已有合作企业": "1. 学校支撑……\n2. 企业合作……"
  }
}
```

### tableRows（追加数据行）

处理带固定表头行+空数据行的表格：

```json
{
  "tableRows": {
    "专利号/登记号": [
      ["实用新型专利", "一种xxx装置", "ZL 2024xxxxxxxx.x", "2024-01-01"],
      ["发明专利", "一种xxx方法", "CN 2024xxxxxxxx.x", "2024-06-01"]
    ]
  }
}
```

key 匹配表内任意文字（InnerText.Contains），保留表头行，用提供的二维数组替换数据行。

### 两种模式可并存

```json
{
  "cellReplace": { "项目名称": "xxx" },
  "tableRows": { "专利号": [["类型","名称","专利号","日期"]] }
}
```

## 关键规则

| 规则 | 说明 |
|------|------|
| 标签-值表 | 匹配到某格的 key 后，检测**右方是否有格子**。有 → 填右格；无 → 填本格 |
| 未匹配的表 | 原封保留（承诺书、学院意见、签名日期行 → 不动） |
| `\n` 转换 | JSON 中的 `\n` 自动转为 Word 换行符 |
| 行高自适应 | 填充后自动移除固定行高，长文本不裁切 |
| 字体保留 | 填入文字的字体和字号继承原模板格式 |
| 空格敏感 | 模板中 "姓 名"（有空格）匹配 JSON key "姓 名" |

## 示例：专利 → 申报书

```bash
# 1. 读取模板结构
nong word read 申报书模板.docx --json

# 2. 准备 data.json（参考上述格式）

# 3. 填入
nong word fill 申报书模板.docx data.json -o 申报书-已填.docx --json

# 4. 验证
nong word read 申报书-已填.docx --json
```

## 已知局限

- 承诺函等包含多段结构的表格，cellReplace 不覆盖（无匹配键），原封保留
- 图片不能通过 JSON 填入（需用 `{{tag}}` + `MiniWordPicture`）
- 合并单元格的列宽可能偏移（Word 合并格在 OpenXML 中为多 Cell 模型）
