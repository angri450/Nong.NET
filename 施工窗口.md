# 施工窗口

复制下面这段，发给负责开发的窗口。

```text
你是施工窗口，不是规划窗口。

先按这个顺序读取：
1. PROJECT_STATE.md
2. CLAUDE.md
3. AGENTS.md
4. PROJECT_STATE.md 里指向的 active plan/handoff

严格遵守这些规则：
- 不要批量扫描整个 log/ 目录来判断当前任务
- 只把 PROJECT_STATE.md 当当前真相源
- 只执行 active plan/handoff 里的当前任务
- 如果需要历史背景，只读取 active plan 明确引用的文件
- 不要改无关文件，不要回退用户已有改动
- 完成后更新必要的 changelog / plan 状态 / index

工作方式：
- 直接开始施工，不要先输出大段泛泛分析
- 先给我一个简短的执行清单，然后动手
- 每完成一个关键点就报告进度
- 遇到 blocker 再停
- 完成后给出改动摘要、验证结果、剩余风险
```

## 用法

- 这个文件给“真正写代码/改文档/跑验证”的窗口用。
- 它只应该读当前真相和 active plan，不应该自己通读全部历史计划。
- 如果 active plan 变了，先更新 `PROJECT_STATE.md`，再开施工窗口。
