文献检索管线 — 完全指南

一句话说清楚：你把研究方向写成一条搜索条件，管线自动去五个学术网站搜，去重、排序、生成报告。零 Python、零浏览器、零手工复制粘贴。

1. 核心概念：候选 → 去重 → 返回

管线处理文献分三步。数字越大，结果越精：

步骤干了什么数量相当于
候选 五个网站各自搜回来的原始结果，还没去重、没筛选 几百篇 你在 Google 搜了一堆链接，还没点开看
去重 同一篇论文被多个网站搜到，DOI 相同就只保留一份 几十篇 删掉重复的标签页
返回 按你的搜索条件精确过滤，再按引用数+时效+关键词匹配排序，截取前 N 篇 N 篇 精读清单

你控制数量：--limit 5 返回 5 篇，--limit 50 返回 50 篇。默认 5。

注意：管线不会帮你"写论文"，它不是 ChatGPT。它是帮你跳过"找文献"这个体力活。

2. 搜索条件怎么给

方式一：直接写关键词（最简单）

不用学任何语法，直接写空格分隔的关键词。管线自动识别这是"普通搜索"而不是 DSL，直接传给所有网站。

沸石 缓释 肥料
zeolite slow release fertilizer
适用场景：快速扫一下某个方向有什么文献，不要求精确过滤。中文关键词会自动走秘塔和 AMiner，英文走 OpenAlex 和 Crossref。

方式二：DSL 精确检索

当你知道具体要什么，用类 CNKI 语法精确控制。

三个符号就够了：

符号意思例子
+或者 (OR)'A'+'B' → A 或 B 都行
*并且 (AND)'A'*'B' → 必须同时有 A 和 B
()分组('A'+'B')*'C' → (A或B) 并且必须有 C

实际例子（来自沸石项目的真实检索式）：

# 方向一：沸石基缓释肥料
SU=('沸石'+'斜发沸石'+'分子筛')*('缓释'+'控释'+'慢释')*('肥料'+'化肥'+'农肥')

# 方向二：土传病害抑菌
SU=('土传病害'+'枯萎病'+'根腐病')*('沸石'+'膨润土'+'蒙脱石')*('抑菌'+'抗菌'+'防病')

# 方向三：根际微生态
SU=('根际'+'根区')*('微生物群落'+'微生物多样性')*('沸石'+'矿物'+'改良剂')

# 方向四：采前处理→保鲜
SU=('采前'+'叶面喷施')*('采后'+'保鲜'+'贮藏')*('果实'+'果蔬')

SU 是什么：SU 表示在标题、摘要、关键词里同时找。大部分情况下 SU 就够了。如果需要限定只在标题里找，用 TI 替代 SU。

字段含义适用场景
SU主题 (Title + Abstract + Keywords + Concepts)绝大多数情况
TI标题只搜标题
AU作者找某个人
YE年份限定年份，如 YE BETWEEN(2020,2025)
JN期刊名限定某本期刊
DOI文献唯一号精确找某一篇

4. 命令速查

命令干什么示例
lit parse 检查 DSL 语法对不对 nong lit parse --query "SU='沸石'*'缓释'" --json
lit plan 看管线会生成哪些查询发给网站 nong lit plan --query "SU='沸石'*'缓释'" --json
lit search 执行检索 nong lit search --query "沸石 缓释" --limit 10 --json
lit export 把结果转成文件 nong lit export --input result.json --format markdown -o refs.md
lit batch 扫文件夹搜所有 DSL，生成汇编报告 nong lit batch 搜索文献/ -o 报告.md --limit 10

常用参数

参数作用默认值
--limit返回几篇5
--sources用哪几个源openalex,crossref,aminer,metaso
--modestrict(严格) / recall(宽松)strict
--profile排序偏好: balanced / classic / recentbalanced
-o导出 JSON 文件不导出

实际使用

# 快速搜一下
nong lit search --query "沸石 缓释 肥料" --limit 10 --json

# 精确检索
nong lit search --query "SU='沸石'*'缓释'*'肥料'" --limit 20 --json

# 批量：扫整个文件夹，一锅出报告
nong lit batch C:\搜索文献\ -o C:\报告.md --limit 10

# 把搜索结果导出为 markdown 引用列表
nong lit export --input result.json --format markdown -o 引用.md

5. lit batch — 批量检索工作流

这是最常用的命令。适用于一个项目有多个研究方向，每个方向有一条 DSL 检索式。

前置条件

你有一个文件夹，里面有若干个 .txt 文件，每个文件里写了检索式（一行一条 DSL，或直接写关键词）。

搜索文献的东西/
  ├── 矿物基材料.txt       ← 含两条 DSL
  ├── 土传病害防控.txt     ← 含两条 DSL
  ├── 环境安全性.txt       ← 含一条 DSL
  ├── 采前保鲜.txt         ← 含一条 DSL
  └── 草案.txt             ← 不包含 DSL，自动跳过
nong lit batch 搜索文献的东西/ -o 报告.md --limit 10

生成报告长这样：

# 文献检索报告
生成时间：2026-06-14 12:47
检索源：openalex, crossref, aminer, metaso
结果数/DSL：10

## 矿物基材料
### 沸石缓释体系
**检索式**: SU=('沸石'...)*('缓释'...)*('肥料'...)
候选 99 篇 → 去重 6 篇 → 返回 10 篇

1. **沸石基缓释肥料的研究进展**. 石勤等. 2022. [metaso]
2. **Zeolite-Based Slow-Release Fertilizer**. ...
3. ...
关键规则：bulk 默认用 recall 模式（中文搜英文不丢结果）。每条 DSL 独立跑，互不影响。搜完自动汇总成一份报告。

6. 搜不出来怎么办

现象可能原因怎么解决
中文 DSL 搜不出结果 strict 模式把中文词在英文摘要里比对不过 加 --mode recall 或直接用 lit batch（默认 recall）
返回 0 篇 AND 条件太严格，没有论文同时满足所有关键词 减少 AND 条件（把 * 换成 +），或者把条件拆成多条 DSL 分开发
AMiner 报错 key 过期或格式不对 去 open.aminer.cn 重新生成 JWT token
秘塔报错 key 格式不对或余额用完 确认 key 是 mk- 开头，登录 metaso.cn 查余额
候选很多、返回很少 这是正常的——候选是粗搜，返回是精确筛选 正常现象。实在不够，放宽 DSL 条件

7. 把结果导出给别人看

# 先搜索，同时保存 JSON
nong lit search --query "SU='沸石'*'缓释'" --limit 20 -o result.json --json

# 再导出为各种格式
nong lit export --input result.json --format markdown -o 引用.md
nong lit export --input result.json --format json     -o 引用.json
nong lit export --input result.json --format bibtex   -o 引用.bib

Markdown 格式是 GB/T 7714 引用格式（作者. 标题[J]. 期刊, 年份），可以直接粘进论文或开题报告。

8. 设计原则（为什么这样建）

  1. 宽搜 + 精滤：让搜索引擎尽量多搜（宽搜），拿回来后本地再精确匹配 DSL 条件（精滤）。搜索引擎本身不懂 DSL，不能让它们做精确过滤。
  2. 纯 C#，零外部依赖：五个引擎全部走 HttpClient 直连 REST API。不需要 Python、不需要 MCP、不需要浏览器。
  3. 自动识别纯文本 vs DSL：管线检测输入有没有 =/*/+。有→走 DSL 解析；没有→直接当关键词发给所有引擎。
  4. 中文友好：recall 模式下，中文关键词搜英文论文时不会因为"摘要里没出现这个中文词"就扔掉结果。
