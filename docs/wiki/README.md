# Nong.Cli.Net Wiki

This directory is stable project knowledge for agents and maintainers.

Use it for current architecture, package boundaries, command ownership, and development history. Do not use it for daily execution state; current state belongs in `PROJECT_STATE.md`.

Recommended reading order:

1. `PROJECT_STATE.md`
2. `docs/wiki/architecture.md`
3. `docs/wiki/development-history.md`
4. `docs/wiki/planning-workflow.md`
5. Task-specific module docs or active plan files

The `log/` directory is an archive of plans, changelogs, debug notes, guidance, and generated reports. It is useful evidence, but it is not the current truth source unless `PROJECT_STATE.md` or an active plan links to a specific file.
